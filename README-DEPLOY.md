# Appido — پروژهٔ Next.js 15 (آمادهٔ استقرار)

نسخهٔ پروداکشنِ ماژولارِ صفحهٔ فرود اپیدو. منطق، ظاهر، UI/UX و پالتِ رنگی **مو‌به‌مو** مطابقِ پیش‌نمایشِ تأییدشده است؛ فقط معماری به Next.js منتقل شده تا SEO/AEO، رِندرِ سمتِ سرور و URLِ مجزا برای هر زبان فعال شود.

## پیش‌نیاز
- Node.js نسخهٔ ۱۸٫۱۸ یا بالاتر (پیشنهاد: ۲۰+)
- npm (یا pnpm / yarn)

## اجرا روی سیستمِ خودتان
```bash
npm install
npm run dev      # http://localhost:3000  →  به /fa هدایت می‌شود
```

## بیلدِ پروداکشن
```bash
npm run build    # بیلدِ استاتیک برای هر ۵ زبان (SSG)
npm run start    # اجرای نسخهٔ بیلدشده
```

> توجه: این پروژه در محیطِ تولیدِ کد بیلد گرفته نشده (آن محیط به اینترنت دسترسی نداشت تا پکیج‌ها نصب شوند). کد با دقت و طبقِ الگوی رسمیِ Next.js 15 نوشته شده؛ لطفاً یک‌بار `npm install && npm run build` را اجرا کنید تا تأیید نهایی شود.

## استقرار (Vercel — پیشنهادی)
1. ریپو را به Vercel وصل کنید (یا `vercel` در ترمینال).
2. تنظیماتِ پیش‌فرضِ Next.js به‌صورت خودکار شناسایی می‌شوند.
3. دامنه را در Vercel وصل کنید.
- تشخیصِ خودکارِ کشور (geo) روی Vercel از طریقِ هدرِ `x-vercel-ip-country` کار می‌کند (IR→fa، RU→ru، TR→tr، کشورهای عربی→ar، بقیه→en).

## دو نکتهٔ پیکربندی
1. **آدرسِ دامنه:** در `app/[lang]/layout.tsx` مقدارِ `SITE_URL` را به دامنهٔ واقعی‌تان تغییر دهید (برای canonical و hreflang).
2. **اتصالِ CRM (اختیاری):** متغیرِ محیطیِ `CRM_WEBHOOK_URL` را ست کنید تا رویدادها به CRM/n8n/وب‌هوکِ شما فرستاده شوند (اندپوینت: `POST /api/track`).

## ساختارِ پروژه
```
app/
  [lang]/
    layout.tsx     ← لِی‌اوتِ ریشه: <html lang/dir>، فونت‌ها، متادیتا، hreflang، SSG
    page.tsx       ← صفحه (سمتِ سرور)، چینشِ بخش‌ها
  api/track/route.ts
  globals.css      ← کلِ استایلِ پیش‌نمایش، عیناً (پالت در بالای فایل)
components/
  icons.tsx        ← مجموعهٔ آیکن‌ها (SVG → کامپوننت)
  ThemeScript.tsx  ← ستِ تمِ بدونِ پرش (no-flash)
  ThemeToggle.tsx  ← کلیدِ تم (کلاینت)
  LanguageSwitcher.tsx ← منوی زبان (کلاینت) — ناوبریِ URLِ هر زبان
  Header.tsx
  sections/        ← Hero, Logos, Features, Proof, Pricing, FinalCta, Footer
lib/
  i18n.ts          ← زبان‌ها، RTL، جهت، LANGS
  dictionaries/    ← en, fa, ar, tr, ru (هر زبان فایلِ جدا) + types + loader
middleware.ts      ← هدایتِ مسیرهای بدونِ زبان به /[locale]
```

## سه تغییرِ عمدیِ نامرئی (نسبت به پیش‌نمایش، تأییدِ جابز)
این‌ها هیچ تأثیرِ بصری/UX ندارند و برای نسخهٔ واقعی لازم‌اند:
1. **حذفِ بنرِ «پیش‌نمایش»** بالای صفحه (روی دامنهٔ واقعی بی‌معنا بود). استایلش حذف شده، چون عنصرش وجود ندارد.
2. **تعویضِ زبان = ناوبریِ URL** (`/en`, `/fa`, …) به‌جای استیتِ کلاینتی — همین چیزی‌ست که SEO/AEO را ممکن می‌کند. ظاهر و رفتارِ منو دقیقاً یکی است (همان دکمه‌ها).
3. **ماندگاریِ تم در localStorage + اسکریپتِ no-flash** (پیش‌نمایش این را نداشت). رفتارِ دیده‌شده یکسان است؛ پیش‌فرض «روشن».

افزودنِ معناییِ واحد: بخش‌های بدنه داخلِ `<main>` قرار گرفته‌اند (برای دسترس‌پذیری و SEO) — بدونِ هیچ اثرِ بصری.
